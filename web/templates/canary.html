Canary Tokens

Settings

Canary tokens are enabled by default for all new clients. When a client registers, Duckway auto-generates decoy credentials via canarytokens.org. If an attacker steals and uses them, you get an email alert.

If none selected, all types are deployed by default.

API Tokens via canarytokens.org
Local Decoy Files self-generated with DNS canary
Clients that should NOT receive canary tokens.

How It Works

TypeCategoryWhat gets deployedDeploy Path

API tokens call canarytokens.org to generate real tripwire credentials. Local decoy files are self-generated with embedded DNS canary hostnames — when an attacker resolves the hostname (e.g., by curling a URL in the file), you get alerted.

All canary files are placed at paths that look realistic but use .bak suffixes or non-standard locations. They are NOT in keys.env, so AI agents use the real placeholder keys and ignore these decoys.
