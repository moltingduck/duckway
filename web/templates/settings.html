Gateway Configuration

Configure how clients connect to the Duckway gateway. These values are used in setup commands and client config.

The URL agents use to reach the gateway. Used in install commands and client init. Leave empty to use the admin panel URL (combined mode).
Suggested port for duckway proxy on client machines. Default: 18080.

ModeCompose FilePorts
Split (recommended) docker-compose.prod.yml Admin :9091 + Gateway :8080
Combined docker-compose.combined.yml Everything :8080

In split mode, agents only access the gateway. Admin panel is on a separate port protected by Tailscale ACL.
